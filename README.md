# Todo List

Checklist to manage and track your tasks.

[Get Started!](https://nirmalsubedi.github.io/odin-to-do-list/)

## How to use?

Start by opening the link above.

**Add Task:**

1. Click the `Add Task` button.

2. Enter the task information: title (name), description, due date (date and time), whether to prioritize, and any notes in the modal.

3. Click the `Save` button.

**Close Task Information Dialog:**

Click the `X` button top right / Press ESC.

**Complete Task:**

Click the task / Press space while focusing the checkbox.

**Edit Task:**

1. Click the `Edit Tasks` button to show the controls.

2. Click the Pencil Icon right of the task to open modal.

3. Click the `Save` button after making the edit.

**Delete Task:**

1. Click the `Edit Tasks` button to show the controls.

2. Click the Trash Icon right of the task.

**Delete Completed Tasks:**

1. Click the `Edit Tasks` button to show the controls.

2. Click the `Delete Checked` button to the right.

**Add Project:**

1. Click `Add` button in the sidebar.

2. Enter project name (whitespaces are ignored).
3. Click anywhere / Press Enter.

**Switch Project:**

Click the project name in the sidebar.

**Delete Project:**

1. Click `Edit` button in the sidebar.

2. Click the trash icon to the right of the project name.


## Which skills were demonstrated?

- Using the web storage API to store and retrieve data. 

- Incorporating external library from npm to format dates. 

- Serializing and Deserializing data with JSON.

- Applying the single responsibility principle to split functionality.

- Using the open/closed principle to open the document click listener for extension, whilst keeping it closed for modification.

- Make use of dependency inversion principle on the document click handler to invoke the function of the element clicked.

- Loosely coupling the code responsible for app logic from the one responsible for user interface.
